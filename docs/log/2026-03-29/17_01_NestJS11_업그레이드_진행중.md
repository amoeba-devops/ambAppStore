# NestJS 11 업그레이드 작업 로그

- **시간**: 2026-03-29 17:xx
- **상태**: 작업 중단 (블로커 미해결)

## 요청 사항
- Docker 빌드 시 `@nestjs/swagger@^11.2.6`이 `@nestjs/common@^11.0.1`을 요구하여 NestJS v10 → v11 업그레이드 진행

## 수행한 작업
1. 3개 backend (`platform`, `app-car-manager`, `app-stock-management`) package.json의 @nestjs/* 패키지 v10 → v11 업데이트
2. 각 backend별 `npm install` 성공 (local node_modules에 v11 설치됨)
3. root package.json에 `overrides` 추가 시도

## 변경된 파일
- `apps/platform/backend/package.json` — @nestjs/* ^10 → ^11
- `apps/app-car-manager/backend/package.json` — @nestjs/* ^10 → ^11
- `apps/app-stock-management/backend/package.json` — @nestjs/* ^10 → ^11, @nestjs/schedule ^4 → ^6.1
- `package.json` (root) — `overrides` 섹션 추가

## 미해결 이슈
- **Root hoisting 문제**: npm workspace가 root `node_modules/@nestjs/common`을 v10.4.22로 hoisting
- `overrides`로 강제 v11 지정했으나 효과 없음
- 이로 인해 `tsc --noEmit` 시 DynamicModule 타입 충돌 발생

3. Docker 빌드 테스트
4. Staging Nginx에 stock-management 라우트 추가
