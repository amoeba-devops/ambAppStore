## 2. Changes Summary (변경 사항 요약)

### Phase 1: Backend — POST /api/v1/trip-logs

| 파일 | 변경유형 | 내용 |
|------|----------|------|
| `domain/trip-log/dto/request/create-trip-log.request.ts` | **신규** | Request DTO — 4 필수 + 14 선택 필드, class-validator 데코레이터 |
| `domain/trip-log/service/trip-log.service.ts` | 수정 | `create()` 메서드 추가 — proxy DispatchRequest 생성 + TripLog 생성 |
| `domain/trip-log/controller/trip-log.controller.ts` | 수정 | `POST /trip-logs` 엔드포인트 추가 — `@Auth()`, `@ApiOperation` |

**핵심 로직**:
- `cdr_id` NOT NULL 제약 → proxy DispatchRequest 자동 생성 (`cdrIsProxy=true`, `cdrStatus=COMPLETED`, `cdrNote='MANUAL_ENTRY'`)
- import-orchestrator 기존 패턴과 동일한 접근 방식
- 거리 자동 계산: `odo_end - odo_start` (둘 다 입력 시)

### Phase 2: Frontend — TripLogFormPage

| 파일 | 변경유형 | 내용 |
|------|----------|------|
| `services/api.ts` | 수정 | `tripLogApi.create()` 메서드 추가 |
| `hooks/useTripLogs.ts` | 수정 | `useCreateTripLog()` mutation hook 추가 |
| `pages/TripLogFormPage.tsx` | 수정 (전체 재작성) | Zod schema + React Hook Form + 5개 섹션 폼 |

**폼 구성**:
- 기본 정보: 차량 드롭다운, 운전자 드롭다운, 출발지*, 목적지*
- 운행 정보: 출발/도착일시 (datetime-local), 출발/도착 주행거리
- 비용 정보: 주유 체크박스 → 조건부 주유량/주유비, 통행료
- 고객/문서: 고객명, Bill No, CDF No
- 기타: 사고여부, 메모, 용도 코드, 업무 사용 비율

### Phase 3: i18n

| 파일 | 변경유형 | 내용 |
|------|----------|------|
| `i18n/locales/ko/car.json` | 수정 | `tripLog` 섹션 22개 키 + `common.select` 추가 |
