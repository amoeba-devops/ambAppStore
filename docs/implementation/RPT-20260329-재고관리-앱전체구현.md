## 1. 작업 요약

재고관리 앱(app-stock-management)의 전체 초기 구현을 완료하였다. 백엔드 NestJS BFF와 프론트엔드 React SPA를 처음부터 구현하고, Docker 컨테이너 환경 및 DB 초기화 스크립트까지 작성하였다.

### 핵심 구현 사항
- **Dual Track 인증**: AMA SSO + Direct Login (자체 발급 JWT)
- **18개 DB 테이블**: Corporation, User, UserApplication, Product, SKU, SkuIdCode, Channel, Inventory, Transaction, ReceivingSchedule, SalesOrder, OrderBatch, Forecast, SafetyStock, Parameter, SeasonalityIndex, WeeklyAggregation, MonthlyAggregation
- **주간 배치 파이프라인**: 수요예측(SMA×SI) → 안전재고(Z×σ×√(LT+RP)) → 발주 제안(MOQ 보정)
- **19개 프론트엔드 페이지**: 인증 3개 + 도메인 16개

## 3. 아키텍처 요약

### 3.1 인증 흐름 (Dual Track)

```
Track 1 (AMA SSO):
  AMA JWT → POST /api/v1/auth/ama-exchange → Internal JWT

Track 2 (Direct Login):
  Entity Code → POST /api/v1/entities/validate → 200
  Email + Password → POST /api/v1/auth/login → Internal JWT
  (tempPassword) → POST /api/v1/auth/change-password → 비밀번호 변경
```

### 3.2 핵심 비즈니스 로직

```
ATS = inv_current_qty − inv_pending_shipment_qty

Safety Stock = Z × σ × √(LT + RP)
  Z = service level → z-score (95% → 1.645)
  σ = 주간 수요 표준편차
  LT = lt1 + lt2 + lt3 + lt4 + lt5 (일)
  RP = review period (주 → 일)

Forecast = SMA(N주) × SI(해당 월)

Order Proposal = max(Safety_Stock + Forecast − ATS, 0)
  → MOQ 보정: ceil(qty / MOQ) × MOQ
```

### 3.3 배치 파이프라인

```
@Cron('0 0 1 * * 1')  // 매주 월요일 01:00
Pipeline:
  1. weeklyAggregate()    → asm_weekly_aggregations
  2. monthlyAggregate()   → asm_monthly_aggregations
  3. computeForecasts()   → asm_forecasts (SMA × SI)
  4. computeSafetyStocks()→ asm_safety_stocks (Z×σ×√(LT+RP))
  5. generateOrderProposals() → asm_order_batches (MOQ 보정)
```
