## 2. Implementation Plan / 단계별 구현 계획

### Phase 1: Backend API (3 files)

**Step 1** — Response DTO 생성  
- `entity-subscription.response.ts` 신규 생성  
- └─ 사이드 임팩트: 없음

**Step 2** — Service에 `findByEntity(entId)` 메서드 추가  
- `subscription.service.ts` 수정  
- AppService의 `findAll()` + SubscriptionRepository 조인 쿼리  
- └─ 사이드 임팩트: 기존 메서드에 영향 없음

**Step 3** — Controller에 `GET entity/:entId` 엔드포인트 추가  
- `subscription.controller.ts` 수정  
- 인증 불필요 (Public)  
- └─ 사이드 임팩트: 기존 라우트와 충돌 없음 (`entity/:entId`는 새 경로)

**Step 4** — CORS에 스테이징 도메인 추가  
- `main.ts` 수정  
- └─ 사이드 임팩트: 스테이징 도메인에서의 API 호출 허용
