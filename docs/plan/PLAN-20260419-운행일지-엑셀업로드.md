## 2. 단계별 구현 계획

### Phase 1: 스키마 확장 (BE Entity + DB)

**Step 1-1: car_trip_logs 엔티티에 물류 컬럼 추가**
- `ctl_customer_name` VARCHAR(100) NULL
- `ctl_bill_no` VARCHAR(100) NULL
- `ctl_cdf_no` VARCHAR(200) NULL
└─ 사이드 임팩트: 기존 API 응답에 null 필드 추가 (하위 호환)

**Step 1-2: TripLogFeeEntity 신규** (`car_trip_log_fees`)
- 1:N 관계 (trip_log → fees)
- ENUM: FUEL, TOLL, PARKING, WASH, TIRE, MAINTENANCE, OTHER
└─ 사이드 임팩트: 없음 (신규 테이블)

**Step 1-3: ImportLogEntity 신규** (`car_import_logs`)
- 업로드 이력 추적
└─ 사이드 임팩트: 없음 (신규 테이블)

**Step 1-4: Mapper/Response 업데이트**
- TripLogResponse에 customerName, billNo, cdfNo 추가
- TripLogMapper에 매핑 추가

### Phase 2: Import 파이프라인 (BE)

**Step 2-1: exceljs 설치 + ExcelParserService**
- `.xls/.xlsx` 로드 → 행 배열 반환
- 헤더 행 자동 인식 (row 10~12)

**Step 2-2: 매핑 프로파일 CR-Vietnam-Truck-v1**
- 시간 파서: `6h50` → `06:50`, `20h20\` → `20:20`
- 날짜 Forward-Fill
- 기타비 코드 분류기 (XANG, BAI, RUA XE, VA VO, CUNG XE)

**Step 2-3: ImportOrchestrator**
- Dry-run 모드: 검증만 수행, DB 저장 안 함
- 실행 모드:
  1. Synthetic Dispatch 생성 (per row)
  2. TripLog 생성
  3. Fee 레코드 생성
  4. 정비성 비용 → MaintenanceRecord 생성
- 멱등키 검사: (cvh_id + depart_actual + odo_start)
- 행 단위 트랜잭션 (부분 실패 허용)

**Step 2-4: ImportController**
```
POST /api/v1/trip-logs/import    (multipart, @Auth)
  Query: ?dryRun=true|false
  Body: file, vehicle_id, driver_id, profile
```

### Phase 3: 프론트엔드

**Step 3-1: TripLogImportModal**
- 차량 선택 (기존 useVehicles)
- 운전자 선택 (기존 useDrivers)
- 프로파일 선택 (CR-Vietnam-Truck-v1)
- 파일 업로드 (multipart/form-data)
- Dry-run 결과 테이블

**Step 3-2: TripLogListPage에 "엑셀 업로드" 버튼 추가**

## 5. DB 마이그레이션

### 스테이징: 자동 (synchronize: true)

### 프로덕션: 수동 SQL 필요
```sql
-- 1. car_trip_logs 컬럼 추가
ALTER TABLE car_trip_logs
  ADD COLUMN ctl_customer_name VARCHAR(100) NULL AFTER ctl_destination,
  ADD COLUMN ctl_bill_no VARCHAR(100) NULL AFTER ctl_customer_name,
  ADD COLUMN ctl_cdf_no VARCHAR(200) NULL AFTER ctl_bill_no;

-- 2. car_trip_log_fees 테이블 생성
CREATE TABLE car_trip_log_fees (
  ctlf_id CHAR(36) NOT NULL PRIMARY KEY,
  ctl_id CHAR(36) NOT NULL,
  ent_id CHAR(36) NOT NULL,
  ctlf_type ENUM('FUEL','TOLL','PARKING','WASH','TIRE','MAINTENANCE','OTHER') NOT NULL,
  ctlf_amount INT NOT NULL,
  ctlf_currency CHAR(3) NOT NULL DEFAULT 'VND',
  ctlf_note VARCHAR(200) NULL,
  ctlf_created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_ctlf_ctl (ctl_id),
  FOREIGN KEY (ctl_id) REFERENCES car_trip_logs(ctl_id)
);

-- 3. car_import_logs 테이블 생성
CREATE TABLE car_import_logs (
  cil_id CHAR(36) NOT NULL PRIMARY KEY,
  ent_id CHAR(36) NOT NULL,
  cil_filename VARCHAR(255) NOT NULL,
  cil_profile VARCHAR(50) NOT NULL,
  cil_total_rows INT NOT NULL DEFAULT 0,
  cil_success_cnt INT NOT NULL DEFAULT 0,
  cil_fail_cnt INT NOT NULL DEFAULT 0,
  cil_status ENUM('PROCESSING','DONE','FAILED') NOT NULL DEFAULT 'PROCESSING',
  cil_uploaded_by CHAR(36) NOT NULL,
  cil_created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  cil_updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_cil_ent (ent_id)
);
```
