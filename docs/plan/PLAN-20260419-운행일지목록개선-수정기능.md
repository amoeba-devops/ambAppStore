## 1. 시스템 개발 현황 분석

### 현재 상태
- **목록 컬럼**: 차량번호 → 날짜 → 노선 → 주행거리 → 주유비 → 상태 (6개)
- **필터**: 월 선택기만 존재, 차량/노선 필터 없음
- **수정**: 미구현 — `useUpdateTripLog` 훅은 존재하나 UI에서 미사용
- **Mapper 문제**: driver 관계는 JOIN하지만 매핑 안 됨, dispatch 관계는 findAll에서 JOIN 안 됨
- **Response 누락**: `driverName`, `passengerCount` 필드 없음

### 제약사항
- 탑승인원은 `dispatchRequest.cdrPassengerCount`에 저장됨 → trip log update 시 dispatch도 함께 업데이트 필요
- VERIFIED 상태는 수정 불가 (기존 비즈니스 규칙)

## 2. 단계별 구현 계획

### Phase 1: Backend — Response 보강 + Update 확장

**Step 1-1**: TripLogResponse에 `driverName`, `passengerCount` 추가
- 파일: `dto/response/trip-log.response.ts` (수정)
- 필드: `driverName: string | null`, `passengerCount: number | null`
- └─ 사이드 임팩트: 없음 (기존 필드 변경 없음, 추가만)

**Step 1-2**: TripLogMapper에 driver/dispatch 매핑 추가
- 파일: `mapper/trip-log.mapper.ts` (수정)
- `driverName ← entity.driver?.cvdDriverName`
- `passengerCount ← entity.dispatchRequest?.cdrPassengerCount`
- └─ 사이드 임팩트: 없음

**Step 1-3**: Service.findAll에 dispatchRequest JOIN 추가
- 파일: `service/trip-log.service.ts` (수정)
- `.leftJoinAndSelect('t.dispatchRequest', 'dr')` 추가
- └─ 사이드 임팩트: Low — 쿼리 성능 (LEFT JOIN 추가)

**Step 1-4**: UpdateTripLogRequest에 `driver_id` 필드 추가
- 파일: `dto/request/trip-log.request.ts` (수정)
- `@IsUUID() @IsOptional() driver_id?: string`
- └─ 사이드 임팩트: 없음

**Step 1-5**: Service.update에 driver_id 매핑 + 탑승인원 업데이트 추가
- 파일: `service/trip-log.service.ts` (수정)
- `driver_id → tripLog.cvdId` 매핑 추가
- `passenger_count` → dispatch의 `cdrPassengerCount` 업데이트 (trip log의 dispatch relation을 통해)
- └─ 사이드 임팩트: Low — dispatch 테이블 업데이트 추가

### Phase 2: Frontend — 목록 컬럼 + 필터

**Step 2-1**: TripLogListPage 컬럼 변경 + 필터 추가
- 파일: `pages/TripLogListPage.tsx` (수정)
- 컬럼: 날짜 → 차량번호 → 노선 → 주행거리 → 상태
- 차량번호 필터 드롭다운 (useVehicles 훅 사용)
- 노선 텍스트 검색 (client-side filter)
- └─ 사이드 임팩트: 없음

### Phase 3: Frontend — 인라인 수정

**Step 3-1**: LogDetailRow 인라인 편집 모드 구현
- 파일: `components/triplog/LogDetailRow.tsx` (수정)
- 읽기/편집 모드 토글
- 편집 필드: 배정운전자(드롭다운), 탑승인원, 출발거리계, 도착거리계, 주행거리(자동), 주유량, 통행료
- 저장/취소 버튼
- `useUpdateTripLog` 훅 호출
- useDrivers 훅으로 운전자 드롭다운 데이터 로딩
- └─ 사이드 임팩트: 없음
