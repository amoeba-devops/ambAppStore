## 1. 시스템 개발 현황 분석

### 현재 상태
- Backend: 운행일지 생성은 배차 연동(`createFromDispatch`) 또는 Excel 업로드(`import-orchestrator`)로만 가능
- Frontend: `TripLogFormPage.tsx` stub만 존재 (placeholder UI)
- Entity: `cdr_id` NOT NULL → proxy dispatch 생성 필수 (import-orchestrator 패턴 존재)

### 제약사항
- `DispatchRequestEntity` 의존 → trip log 생성 시 proxy dispatch 자동 생성
- `DispatchModule`과 순환 참조 → `forwardRef` 사용 중 (trip-log.module.ts)
- `DispatchRequestEntity`는 이미 trip-log 모듈에서 `TypeOrmModule.forFeature`로 import 중

## 2. 단계별 구현 계획

### Phase 1: Backend — Create API

**Step 1-1**: `CreateTripLogRequest` DTO 생성
- 파일: `dto/request/create-trip-log.request.ts` (신규)
- 필드: `vehicle_id`(필수), `driver_id`(필수), `origin`(필수), `destination`(필수) + UpdateTripLogRequest 필드 전부
- └─ 사이드 임팩트: 없음

**Step 1-2**: `TripLogService.create()` 메서드 추가
- 파일: `service/trip-log.service.ts` (수정)
- 로직: proxy DispatchRequest 생성 → TripLog 생성 (import-orchestrator 패턴)
- └─ 사이드 임팩트: DispatchRequestEntity repo 주입 필요

**Step 1-3**: `TripLogController.create()` 엔드포인트 추가
- 파일: `controller/trip-log.controller.ts` (수정)
- `POST /trip-logs` + `@Auth()` + `@ApiOperation`
- └─ 사이드 임팩트: 없음

### Phase 2: Frontend — Form 구현

**Step 2-1**: `tripLogApi.create` + `useCreateTripLog` 추가
- 파일: `services/api.ts`, `hooks/useTripLogs.ts` (수정)
- └─ 사이드 임팩트: 없음

**Step 2-2**: `TripLogFormPage.tsx` 전체 재작성
- React Hook Form + Zod schema
- 차량/운전자 드롭다운 (useVehicles, useDrivers)
- 폼 섹션: 기본정보 / 운행정보 / 비용정보 / 고객정보 / 기타
- └─ 사이드 임팩트: 없음

### Phase 3: i18n

**Step 3-1**: 번역 키 추가 (ko/en/vi)
- `tripLog.formTitle`, `tripLog.selectVehicle`, `tripLog.selectDriver`, `tripLog.customerName`, `tripLog.billNo`, `tripLog.cdfNo`, `tripLog.refueled`, `tripLog.hasAccident`, `tripLog.note`, `tripLog.krPurposeCode`, `tripLog.krBusinessRatio`, `tripLog.purposeBusiness`, `tripLog.purposeCommute`, `tripLog.purposeOther`
- └─ 사이드 임팩트: 없음
