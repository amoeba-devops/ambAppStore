### Phase 2 — VehicleEditPage 구현

#### Step 2.1: `pages/VehicleEditPage.tsx` 신규 작성
- 의존성:
  - `useParams<{ id: string }>()` — URL 파라미터
  - `useVehicle(id)` — 기존 데이터 로드
  - `useUpdateVehicle()` — PATCH 호출
  - `useForm` + `zodResolver` — 폼 상태
  - `useToastStore.showToast` — 성공/실패 피드백
  - `useNavigate` — 저장/취소 시 상세 페이지 복귀
- 구조:
  1. URL `id` 획득 → `useVehicle` 로 상세 조회
  2. 로딩 중: 로딩 인디케이터
  3. 데이터 없음: "차량을 찾을 수 없습니다"
  4. 데이터 있음: `defaultValues` 로 폼 초기화 (React Hook Form `reset()` 사용)
  5. 수정 불가 필드는 disabled `<input>` 으로 표시 (참고용)
  6. 수정 가능 필드는 편집 가능 `<input>` / `<select>` / `<textarea>`
  7. 제출 핸들러: `updateMutation.mutateAsync({ id, data })` → 성공 toast → navigate
- Zod 스키마 (모두 optional, partial update):
  ```ts
  const editSchema = z.object({
    color: z.string().max(30).optional(),
    displacement: z.coerce.number().int().optional().or(z.literal('').transform(() => undefined)),
    transmission: z.enum(['AUTO', 'MANUAL']).optional(),
    max_passengers: z.coerce.number().int().min(1).optional(),
    max_load_ton: z.coerce.number().optional().or(z.literal('').transform(() => undefined)),
    cargo_type: z.enum(['GENERAL', 'REFRIGERATED', 'HAZARDOUS']).optional().or(z.literal('').transform(() => undefined)),
    purchase_type: z.enum(['OWNED', 'LEASED', 'RENTED']).optional().or(z.literal('').transform(() => undefined)),
    insurance_expiry: z.string().optional(),
    inspection_date: z.string().optional(),
    note: z.string().optional(),
  });
  ```
- **주의**: 응답 DTO는 camelCase(`maxLoadTon`, `insuranceExpiry`), 요청 DTO는 snake_case. 폼 필드명은 snake_case 로 통일(요청 DTO와 정렬).
- 제출 시 빈 문자열 / `undefined` 는 payload 에서 제외 (의도치 않은 null 덮어쓰기 방지)
- └─ **사이드 임팩트**: 없음 (신규 파일)

#### Step 2.2: `App.tsx` 라우트 등록
- Line 14 근처에 import 추가:
  ```tsx
  import { VehicleEditPage } from '@/pages/VehicleEditPage';
  ```
- Routes 블록에 `/vehicles/:id` 라우트 **위**에 edit 라우트 추가 (더 구체적인 경로가 우선):
