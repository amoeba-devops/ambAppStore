## 2. 단계별 구현 계획 / Step-by-Step Plan

### Phase 1: Header Entity 정보 표시 (1 Step)

#### Step 1.1 — Header.tsx 수정: iframe 시 Entity Code + Name 표시

**변경 내용**:
- 언어 선택 드롭다운 좌측에 Entity 정보 배지 추가
- 조건: `embedded && isAuthenticated && user?.entityCode`
- UI: `[EntityCode · UserName]` 텍스트 배지 (gray pill)

**코드 위치**: `Header.tsx` — 언어 선택 `<div className="relative group">` 바로 앞

```tsx
{/* Entity info for embedded (iframe) view */}
{embedded && isAuthenticated && user?.entityCode && (
  <span className="rounded-full bg-gray-100 px-3 py-1 text-sm font-medium text-gray-700">
    {user.entityCode} · {user.name}
  </span>
)}
```

└─ **사이드 임팩트**: 없음. 조건부 렌더링으로 기존 UI 변경 없음.

---

### Phase 2: 구독 모달 개선 (2 Steps)

#### Step 2.1 — 모달 스크롤 처리

**변경 내용**:
- 모달 컨테이너에 `max-h-[90vh] overflow-y-auto` 추가
- 기존: `<div className="relative w-full max-w-md rounded-2xl bg-white p-6 shadow-xl">`
- 변경: `<div className="relative w-full max-w-md max-h-[90vh] overflow-y-auto rounded-2xl bg-white p-6 shadow-xl">`

└─ **사이드 임팩트**: 없음. 작은 화면에서만 스크롤 발생, 대형 화면에서는 기존과 동일.

#### Step 2.2 — 신청자 정보 필드 제거 + Zod 스키마 정리

**변경 내용**:

1. **Zod 스키마** — `requester_name`, `requester_email` 필드 삭제
   ```typescript
   // AS-IS
   const schema = z.object({
     ent_id: z.string().min(1),
     ent_code: z.string().min(1).regex(/^[A-Za-z0-9-]+$/),
     ent_name: z.string().min(1).max(100),
     requester_name: z.string().optional(),         // ← 삭제
     requester_email: z.string().email()...         // ← 삭제
     reason: z.string().max(500).optional(),
   });
   
   // TO-BE
   const schema = z.object({
     ent_id: z.string().min(1),
     ent_code: z.string().min(1).regex(/^[A-Za-z0-9-]+$/),
     ent_name: z.string().min(1).max(100),
     reason: z.string().max(500).optional(),
   });
   ```

2. **defaultValues** — `requester_name`, `requester_email` 삭제

3. **JSX** — 신청자 정보 fieldset 전체 삭제 (인증/비인증 모두)
   - 인증 시: `<fieldset>` (이름+이메일 읽기) 삭제
   - 비인증 시: `<fieldset>` (이름+이메일 입력) 삭제

4. **onSubmit** — publicMutation 호출 시 `requester_name`, `requester_email` 미전송

└─ **사이드 임팩트**: 기존 public 구독 API에서 해당 필드가 optional이므로 백엔드 변경 불필요. 유저 추적이 필요하면 JWT에서 추출.
