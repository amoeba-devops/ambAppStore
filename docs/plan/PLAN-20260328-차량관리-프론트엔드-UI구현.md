## 1. 시스템 개발 현황 분석

### 1.1 디렉토리 구조 (현재)

```
apps/app-car-manager/frontend/src/
├── App.tsx                           # 10개 라우트
├── main.tsx / index.css / vite-env.d.ts
├── components/layout/AppLayout.tsx   # 사이드바(w-60, 6 NavLink)
├── hooks/                            # 5개 훅 파일 (query key factory)
├── i18n/                             # i18n.ts + ko/en/vi car.json (88키)
├── lib/                              # api-client.ts + query-client.ts
├── pages/                            # 10개 페이지 컴포넌트
├── services/api.ts                   # 6개 API 모듈
└── stores/auth.store.ts              # Zustand auth 스토어
```

### 1.2 기술 스택

| 항목 | 버전 |
|------|------|
| React | 18 |
| TypeScript | 5 |
| React Router DOM | 6 |
| React Query (TanStack) | 5 |
| Zustand | 5 |
| React Hook Form | 7 |
| Zod | latest |
| i18next | latest |
| lucide-react | latest |
| TailwindCSS | 3 |
| Vite | 5 |

### 1.3 기존 코드 활용 가능 항목

| 항목 | 파일 | 상태 |
|------|------|------|
| API 서비스 레이어 | `services/api.ts` | ✅ 완전 활용 (6개 API 모듈) |
| React Query 훅 | `hooks/*.ts` | ✅ 완전 활용 (5개 파일, 27+훅) |
| API 클라이언트 | `lib/api-client.ts` | ✅ 그대로 유지 |
| Query Client 설정 | `lib/query-client.ts` | ✅ 그대로 유지 |
| Auth 스토어 | `stores/auth.store.ts` | ✅ 그대로 유지 |
| i18n 기본 설정 | `i18n/i18n.ts` | ✅ 유지 (번역키만 추가) |

### 1.4 제약사항

- 백엔드 API 변경 없음 (순수 프론트엔드 작업)
- 기존 훅/서비스 인터페이스 유지 (breaking change 없음)
- TailwindCSS 유틸리티 기반 스타일링 (별도 CSS 모듈 사용하지 않음)
- 모든 UI 텍스트 i18n 처리 필수
