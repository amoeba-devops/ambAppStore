## 테스트 범위

- 대상: DriverFormModal, VehicleDetailPage 운전자 탭, DriverListPage, App 라우팅, AppLayout 사이드바

- 사전 조건: JWT 인증 완료 상태, 차량 1대 이상 등록

## 테스트케이스 목록

### TC-001: 차량 상세 → 운전자 추가 모달 열기

- **연관 요구사항**: R-001
- **사전 조건**: 차량 상세 페이지 진입 → 운전자 탭 선택
- **테스트 단계**:
  1. "운전자 추가" 버튼 클릭
  2. 모달이 열리는지 확인
- **기대 결과**:
  - DriverFormModal 표시
  - 제목: "운전자 추가" (i18n `driverForm.title`)
  - 폼 필드: AMA 사용자 ID (필수), 역할 선택 (필수), 메모 (선택)
  - 버튼: "취소", "등록"
- **우선순위**: 상

### TC-002: 모달 — 필수 필드 미입력 시 제출 방지

- **연관 요구사항**: R-002
- **사전 조건**: TC-001 모달 열림 상태
- **테스트 단계**:
  1. AMA 사용자 ID 빈 상태에서 "등록" 클릭
  2. 역할 미선택 상태에서 "등록" 클릭
- **기대 결과**:
  - 폼 제출되지 않음
  - 필수 필드에 검증 에러 표시
- **우선순위**: 상

### TC-003: 모달 — 잘못된 UUID 형식 검증

- **연관 요구사항**: R-002
- **사전 조건**: TC-001 모달 열림 상태
- **테스트 단계**:
  1. AMA 사용자 ID에 "abc123" (UUID 아닌 값) 입력
  2. 역할 "주운전자" 선택
  3. "등록" 클릭
- **기대 결과**:
  - UUID 형식 검증 에러 메시지 표시
  - API 호출되지 않음
- **우선순위**: 상

### TC-004: 운전자 등록 성공 (차량 배정)

- **연관 요구사항**: R-001, R-002
- **사전 조건**: TC-001 모달 열림, 차량 상세 페이지에서 열었으므로 vehicleId 자동 전달
- **테스트 단계**:
  1. AMA 사용자 ID에 유효한 UUID 입력
  2. 역할 "주운전자 (PRIMARY)" 선택
  3. 메모에 "테스트 운전자" 입력
  4. "등록" 클릭
- **기대 결과**:
  - `POST /api/v1/drivers` 호출 (body: `{ ama_user_id, role: "PRIMARY_DRIVER", vehicle_id, note }`)
  - 모달 닫힘
  - 운전자 목록에 새 운전자 표시 (★ 주운전자 그룹)
  - DriverCard에 역할 뱃지, 상태(ACTIVE) 표시
- **우선순위**: 상

### TC-005: 운전자 등록 성공 (차량 미배정 — DriverListPage)

- **연관 요구사항**: R-002, R-003
- **사전 조건**: `/drivers` 페이지 진입
- **테스트 단계**:
  1. "운전자 등록" 버튼 클릭 → 모달 열림
  2. AMA 사용자 ID 입력, 역할 "풀 운전자 (POOL)" 선택
  3. "등록" 클릭
- **기대 결과**:
  - `POST /api/v1/drivers` 호출 (body: `{ ama_user_id, role: "POOL_DRIVER" }`, vehicle_id 없음)
  - 모달 닫힘
  - 운전자 목록 테이블에 새 행 추가 (배정차량: "-")
- **우선순위**: 상

### TC-006: 중복 운전자 등록 실패 (CAR-E4002)

- **연관 요구사항**: R-002
- **사전 조건**: TC-004에서 등록한 운전자의 동일 ama_user_id 사용
- **테스트 단계**:
  1. 모달에서 동일한 AMA 사용자 ID 입력
  2. 아무 역할 선택 후 "등록" 클릭
- **기대 결과**:
  - API 400/409 응답 (CAR-E4002)
  - 모달에 에러 메시지 표시: "이미 등록된 운전자입니다"
  - 모달 닫히지 않음 (사용자가 수정 가능)
- **우선순위**: 상

### TC-007: 모달 닫기 동작

- **연관 요구사항**: R-001
- **사전 조건**: TC-001 모달 열림 상태
- **테스트 단계**:
  1. "취소" 버튼 클릭 → 모달 닫힘 확인
  2. 다시 열기 → X 버튼 클릭 → 모달 닫힘 확인
  3. 다시 열기 → 오버레이(배경) 클릭 → 모달 닫힘 확인
  4. 다시 열기 → ESC 키 → 모달 닫힘 확인
- **기대 결과**:
  - 4가지 방법 모두 모달 정상 닫힘
  - 닫은 후 폼 상태 초기화
- **우선순위**: 중

### TC-008: 사이드바 운전자 관리 메뉴
