# 풀 운전자 관리 — 테스트케이스

**문서 ID**: TC-20260418-풀운전자관리
**작성일**: 2026-04-18
**참조**: PLAN-20260418-풀운전자관리

---

## 테스트 범위

- 대상: PoolDriverModal, VehicleDetailPage, assign/unassign API
- 환경: 스테이징
- 사전 조건: POOL_DRIVER 1명 이상 등록, 차량 1대 이상

---

### TC-001: 모달 열기

- **사전 조건**: 차량 상세 → 운전자 탭
- **테스트**: "🌐 풀 운전자 관리" 클릭
- **기대**: PoolDriverModal 표시, 배정된/가용 풀 운전자 목록
- **우선순위**: 상

### TC-002: 풀 운전자 배정

- **사전 조건**: 미배정 풀 운전자 존재
- **테스트**: "배정" 버튼 클릭
- **기대**: `PATCH /drivers/:id/assign` → 성공 → 배정된 목록으로 이동
- **우선순위**: 상

### TC-003: 풀 운전자 해제

- **사전 조건**: 현재 차량에 배정된 풀 운전자 존재
- **테스트**: "해제" 버튼 클릭
- **기대**: `PATCH /drivers/:id/unassign` → 성공 → 가용 목록으로 이동
- **우선순위**: 상

### TC-004: 배정 후 운전자 탭 반영

- **사전 조건**: TC-002 배정 완료
- **테스트**: 모달 닫기 → 운전자 탭 확인
- **기대**: 풀 운전자 그룹에 새로 배정된 운전자 표시
- **우선순위**: 상

### TC-005: 풀 운전자 없을 때

- **사전 조건**: 법인 내 POOL_DRIVER 0명
- **테스트**: 모달 열기
- **기대**: "등록된 풀 운전자가 없습니다" 안내 메시지
- **우선순위**: 중

### TC-006: 모달 닫기

- **테스트**: 닫기 버튼 / ESC / 오버레이 클릭
- **기대**: 모달 닫힘
- **우선순위**: 중

### TC-007: i18n 다국어

- **테스트**: KO → EN → VI 전환
- **기대**: 모달 텍스트 정상 번역
- **우선순위**: 하
