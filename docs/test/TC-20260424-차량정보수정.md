## 2. 테스트 케이스

### TC-01. `/vehicles/:id/edit` 진입 (라우트 매칭)
- **전제**: AMA 인증 완료, 대상 차량 조회 가능
- **절차**:
  1. URL `https://stg-apps.amoeba.site/app-car-manager/vehicles/{id}/edit` 접속
- **기대결과**:
  - 빈 화면이 아닌 차량 수정 페이지가 렌더링된다
  - 페이지 헤더에 "차량 정보 수정" (i18n) 또는 차량번호가 표시된다
- **통과 조건**: 폼이 화면에 표시됨

### TC-02. 기존 값 prefill
- **절차**: TC-01 진입
- **기대결과**:
  - 수정 가능 필드(색상/배기량/변속기/최대탑승인원/최대적재량/화물유형/구매유형/보험만료일/검사일/비고)에 기존 값이 채워져 있다
  - 수정 불가 필드(번호판/제조사/모델/연식/VIN/연료/차종)는 disabled 상태로 값이 표시된다
- **통과 조건**: 모든 기존 값이 폼에 반영됨

### TC-03. 필드 편집 후 저장 — 성공 플로우
- **절차**:
  1. 색상 필드 값 변경 (예: `White` → `Silver`)
  2. 비고 필드에 텍스트 입력
  3. "저장" 버튼 클릭
- **기대결과**:
  - `PATCH /api/v1/vehicles/{id}` 요청이 전송됨 (DevTools Network)
  - 요청 body: `{ "color": "Silver", "note": "..." }` (변경된 필드만 또는 전체 필드)
  - 응답 200
  - 성공 toast 표시
  - `/vehicles/{id}` 상세 페이지로 이동
  - 상세 페이지에 변경된 값이 반영됨
- **통과 조건**: 변경사항이 DB에 반영되고 UI에 표시됨

### TC-04. 취소 버튼
- **절차**:
  1. 수정 페이지 진입
  2. 필드 값 변경 (저장 X)
  3. "취소" 버튼 클릭
- **기대결과**:
  - `/vehicles/{id}` 상세 페이지로 이동
  - 변경사항 미반영 (기존 값 유지)
- **통과 조건**: 변경 폐기 후 상세 페이지 복귀

### TC-05. 수정 불가 필드 편집 차단
- **절차**:
  1. 수정 페이지 진입
  2. 번호판/제조사/모델/연식/VIN/연료/차종 필드 편집 시도
- **기대결과**:
  - 해당 필드는 disabled 이므로 편집 불가
  - (또는 hidden input 으로 폼 제출에서 제외됨)
- **통과 조건**: 식별 필드가 변경되지 않음

### TC-06. 잘못된 ID 접근
- **절차**:
  1. URL `/vehicles/00000000-0000-0000-0000-000000000000/edit` 접속
- **기대결과**:
  - `useVehicle` 에러 → "차량을 찾을 수 없습니다" 또는 동등한 에러 UI
- **통과 조건**: 앱이 crash 하지 않고 에러 상태 표시

### TC-07. validation 실패 — 최대 탑승인원 음수
- **절차**:
  1. 수정 페이지 진입
  2. 최대 탑승인원 필드에 `-1` 입력
  3. "저장" 클릭
- **기대결과**:
  - 필드 에러 메시지 표시 (Zod `min(1)` 위반)
  - PATCH 요청 전송 안 됨
- **통과 조건**: 저장 블록 + 에러 표시

### TC-08. 부분 업데이트 (빈 값 처리)
- **절차**:
  1. 수정 페이지 진입
  2. 보험 만료일 필드를 비움 (기존 값 삭제)
  3. "저장" 클릭
- **기대결과**:
  - 빈 값이 `undefined` 로 payload 제외 → 기존 값 유지
  - **또는** 명시적으로 `null` 전송 → DB에서 해당 컬럼 NULL 처리
  - 구현 방식에 따라 예상 동작 일치
- **통과 조건**: 예상 동작(undefined 제외)과 일치

### TC-09. i18n — 한국어
- **절차**: 언어 ko 로 수정 페이지 진입
- **기대결과**: 모든 UI 텍스트가 한국어로 표시
- **통과 조건**: 하드코딩 영문/기타 언어 없음

### TC-10. i18n — 영어
- **절차**: 언어 en 로 수정 페이지 진입
- **기대결과**: 모든 UI 텍스트가 영어로 표시
- **통과 조건**: 동일

### TC-11. i18n — 베트남어
- **절차**: 언어 vi 로 수정 페이지 진입
- **기대결과**: 모든 UI 텍스트가 베트남어로 표시
- **통과 조건**: 동일

### TC-12. 상세 페이지 "수정" 버튼 정상 연결
- **절차**:
  1. `/vehicles/{id}` 접속
  2. "수정" 버튼 클릭
- **기대결과**: `/vehicles/{id}/edit` 로 이동, 빈 화면이 아닌 폼이 렌더링
- **통과 조건**: 원활한 네비게이션

### TC-13. 캐시 invalidation 확인
- **절차**:
  1. 차량 목록 페이지 진입 → 색상 값 확인
  2. 상세 → 수정 페이지 이동 → 색상 변경 → 저장
  3. 상세 페이지로 복귀 후 다시 목록 페이지로 이동
- **기대결과**: 목록 페이지에서도 변경된 색상이 반영됨 (새 쿼리 없이 캐시 업데이트)
- **통과 조건**: `useUpdateVehicle` onSuccess 에서 list/detail invalidate 정상

### TC-14. 타입 체크 & 빌드
- **절차**: `cd apps/app-car-manager/frontend && npm run build`
- **기대결과**: 에러 없이 빌드 성공
- **통과 조건**: 종료 코드 0
