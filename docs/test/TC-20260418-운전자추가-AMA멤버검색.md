## 테스트 범위

- 대상: AMA 멤버 프록시 API (BE), DriverFormModal 검색 UI (FE)
- 환경: 스테이징 (https://stg-apps.amoeba.site/app-car-manager)
- 사전 조건: JWT 인증 완료, AMA 스테이징 서버 정상 가동

## 테스트케이스 목록

### TC-001: BE — AMA 멤버 프록시 API 정상 응답

- **연관 요구사항**: R-001
- **사전 조건**: 인증된 사용자, AMA 서버 정상
- **테스트 단계**:
  1. `GET /api/v1/ama/members` 호출 (Bearer Token 포함)
- **기대 결과**:
  - HTTP 200
  - `{ success: true, data: [{ userId, name, email, ... }] }` 형식
  - 동일 법인(ent_id) 멤버만 반환
- **우선순위**: 상

### TC-002: BE — 검색 파라미터 전달

- **연관 요구사항**: R-001
- **사전 조건**: TC-001 통과
- **테스트 단계**:
  1. `GET /api/v1/ama/members?search=김` 호출
- **기대 결과**:
  - "김"이 포함된 멤버만 반환 (이름 또는 이메일 기준)
  - 빈 결과 시 빈 배열 반환 (`data: []`)
- **우선순위**: 상

### TC-003: BE — 인증 없이 호출 시 401

- **연관 요구사항**: R-001
- **사전 조건**: 없음
- **테스트 단계**:
  1. Authorization 헤더 없이 `GET /api/v1/ama/members` 호출
- **기대 결과**:
  - HTTP 401 Unauthorized
- **우선순위**: 상

### TC-004: BE — AMA 서버 장애 시 에러 처리

- **연관 요구사항**: R-001
- **사전 조건**: AMA API 응답 불가 상황 (또는 잘못된 URL)
- **테스트 단계**:
  1. AMA_API_BASE_URL이 잘못된 상태에서 API 호출
- **기대 결과**:
  - HTTP 502 또는 500
  - `{ success: false, error: { code, message } }` 형식
  - 서버 크래시 없음
- **우선순위**: 중

### TC-005: FE — 모달 열기 시 검색 입력 표시

- **연관 요구사항**: R-002
- **사전 조건**: 차량 상세 → 운전자 탭

- **기대 결과**:

  - placeholder: "이름 또는 이메일 검색..." (i18n)
  - 역할 선택, 메모, 버튼은 기존과 동일
- **우선순위**: 상

### TC-007: FE — 검색 결과 드롭다운 표시
