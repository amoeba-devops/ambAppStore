# BUG-260419: 배차 확정 시 운전자 선택 불가 + 배정정보 미저장

**발견일**: 2026-04-19
**심각도**: Major
**상태**: Fixed
**URL**: https://stg-apps.amoeba.site/app-car-manager/dispatches

---

## 1. 버그 현상

- 배차 확정 모달에서 운전자를 선택해도 반영되지 않음
- 운전자가 "운행중" 표시되어 선택 불가
- 배차 확정 후 배정운전자 정보가 배차 상세에 저장되지 않음

## 2. 원인 분석

### 2.1 DriverSelector 역할 필터 불일치

`DriverSelector` 컴포넌트 (line 19):
```typescript
const ROLE_ORDER = ['PRIMARY', 'SUB', 'POOL'];
```

실제 API 응답의 role 값:
```
'PRIMARY_DRIVER', 'SUB_DRIVER', 'POOL_DRIVER'
```

→ `drivers.filter((d) => d.role === 'PRIMARY')` → **항상 빈 배열** → 운전자 그룹이 표시되지 않음

### 2.2 DriverSelector 상태 필터 불일치

```typescript
const isAvailable = driver.status === 'AVAILABLE';
```

실제 운전자 상태:
```
'ACTIVE', 'ON_LEAVE', 'INACTIVE'
```

→ `isAvailable`이 항상 `false` → 모든 운전자가 disabled/비활성

### 2.3 useAvailableDrivers 데이터

`useAvailableDrivers`는 `GET /drivers/available`을 호출하며, 이는 `ACTIVE` 상태의 미배정 운전자만 반환. 배정된 운전자는 목록에서 제외됨.

## 3. 수정 방안

1. **DriverSelector ROLE_ORDER**: `'PRIMARY'` → `'PRIMARY_DRIVER'` 등으로 변경
2. **DriverSelector isAvailable**: `status === 'AVAILABLE'` → `status === 'ACTIVE'`로 변경
3. **DispatchConfirmModal**: `useAvailableDrivers()` → `useDrivers()`로 변경 (전체 ACTIVE 운전자)
4. **DriverSelector 매핑 제거**: DispatchConfirmModal에서 role을 매핑할 때 `_DRIVER` suffix를 잘라내지 않도록 수정

## 4. 영향 범위

- `components/dispatch/DriverSelector.tsx` — ROLE_ORDER, isAvailable
- `components/dispatch/DispatchConfirmModal.tsx` — useDrivers 변경, role 매핑 수정
