## 1. 서비스 개요 (Service Overview)

### AMA App Store란?

AMA App Store는 AMA(AI Management Assistant) 생태계 위에서 동작하는 **파트너 앱 플랫폼**입니다. 법인(Entity) 단위로 업무에 필요한 다양한 앱을 신청하고 사용할 수 있습니다.

### 제공 앱 목록

| 앱 | 아이콘 | 설명 |
|----|--------|------|
| **법인차량관리** | 🚗 | 법인 보유 차량의 배차, 운행, 정비 기록을 관리합니다 |
| **HS Code Tool** | 📦 | 수출입 상품의 HS Code를 검색하고 관리합니다 |
| **매출리포트** | 📊 | 매출 데이터를 시각화하고 리포트를 생성합니다 |
| **재고관리** | 📈 | 제품 재고 현황을 추적하고 예측합니다 |

### 접속 환경

- **권장 브라우저**: Chrome, Edge, Safari (최신 버전)
- **반응형 지원**: 데스크톱, 태블릿, 모바일
- **인증 방식**: AMA Entity 계정 (별도 회원가입 불필요)

## 2. 시작하기 (Getting Started)

### 2.1 로그인

1. https://apps.amoeba.site 에 접속합니다.
2. 우측 상단의 **[로그인]** 버튼을 클릭합니다.
3. 로그인 화면에서 다음 정보를 입력합니다:

| 입력 항목 | 설명 | 예시 |
|-----------|------|------|
| **Entity ID** | AMA에서 발급된 법인 식별자 | `VN01` |
| **이메일** | AMA 계정 이메일 | `user@company.com` |
| **비밀번호** | AMA 계정 비밀번호 | - |

4. **[로그인]** 버튼을 클릭합니다.

> **참고**: AMA App Store는 AMA SSO 방식을 사용합니다. AMA에서 사용하는 계정으로 동일하게 로그인하면 됩니다.

### 2.2 로그인 후 화면

로그인에 성공하면 상단 헤더에 다음 항목이 표시됩니다:

```
[🏪 AMA App Store]  [🌐 한국어 ▾]  [📋 내 구독 관리]  [🔔]  [사용자명]  [로그아웃]
```

| 항목 | 설명 |
|------|------|
| 🏪 AMA App Store | 클릭 시 메인 페이지로 이동 |
| 🌐 언어 선택 | 한국어 / English / Tiếng Việt 전환 |
| 📋 내 구독 관리 | 신청한 앱 목록 및 상태 확인 |
| 🔔 알림 벨 | 구독 상태 변경 알림 확인 |
| 사용자명 | 현재 로그인한 사용자 이름 |
| 로그아웃 | 세션 종료 |

### 2.3 로그아웃

상단 헤더의 **[로그아웃]** 버튼을 클릭하면 즉시 로그아웃됩니다.

## 3. 앱 둘러보기 (Browsing Apps)

### 3.1 메인 페이지 (앱 카탈로그)

메인 페이지(`/`)에서 등록된 모든 앱을 카드 형태로 확인할 수 있습니다.

각 앱 카드에는 다음 정보가 표시됩니다:
- **앱 아이콘** — 앱을 상징하는 이모지
- **앱 이름** — 앱의 정식 명칭
- **간단 설명** — 앱의 주요 기능 한 줄 요약
- **상태 뱃지** — 현재 앱/구독 상태

### 3.2 상태 뱃지 안내

| 뱃지 | 색상 | 의미 |
|------|------|------|
| **사용가능** | 🟢 초록 | 신청 가능한 앱 |
| **출시예정** | 🟡 노랑 | 아직 출시되지 않은 앱 (신청 불가) |
| **사용중** | 🔵 파랑 | 이미 구독 중인 앱 |
| **심사중** | 🟣 보라 | 신청 후 관리자 심사 대기 중 |
| **만료됨** | 🟠 주황 | 구독 기간이 만료된 앱 |

### 3.3 앱 상세 페이지

앱 카드를 클릭하면 상세 페이지로 이동하여 다음 정보를 확인할 수 있습니다:

- **앱 소개** — 앱의 상세 설명
- **주요 기능** — 주요 기능 목록 (아이콘 + 설명)
- **스크린샷** — 앱 화면 미리보기 (등록된 경우)
- **액션 버튼** — 구독 상태에 따라 다른 버튼 표시

### 3.4 상세 페이지 액션 버튼

| 구독 상태 | 버튼 | 색상 | 동작 |
|-----------|------|------|------|
| 미로그인 | `로그인 후 신청하기` | 회색 (비활성) | - |
| 미신청 | `앱 사용 신청` | 보라색 | 신청 모달 열기 |
| 심사중 | `심사중` | 연보라 (비활성) | - |
| 사용중 | `사용중 — 앱 바로가기` | 파란색 | 앱으로 이동 |
| 만료됨 | `재신청` | 주황색 | 신청 모달 열기 |
| 출시예정 | `출시예정` | 회색 (비활성) | - |

## 4. 앱 사용 신청 (Requesting App Subscription)

### 4.1 신청 절차

1. 메인 페이지에서 원하는 앱 카드를 클릭합니다.
2. 앱 상세 페이지에서 **[앱 사용 신청]** 버튼을 클릭합니다.
3. 신청 모달이 열리면 다음 정보를 입력합니다:

#### 신청 폼 구성

| 섹션 | 필드 | 필수 | 설명 |
|------|------|------|------|
| **앱 정보** | 앱 이름 | 자동 | 선택한 앱 이름 (읽기 전용) |
| **신청자 정보** | 이름 | 자동 | 로그인한 사용자 이름 (읽기 전용) |
| | 이메일 | 자동 | 로그인한 사용자 이메일 (읽기 전용) |
| **Entity 정보** | Entity Code | ✅ | 법인 코드 (예: `AMB-KR`) |
| | Entity Name | ✅ | 법인 이름 (예: `Amoeba Company Korea`) |
| **추가 정보** | 신청 사유 | 선택 | 앱 사용 목적 (최대 500자) |

4. **[신청하기]** 버튼을 클릭합니다.
5. "신청 완료" 메시지가 표시되면 **[확인]** 버튼으로 모달을 닫습니다.

### 4.2 신청 후 절차

```
신청 접수 → 관리자 심사 → 승인/반려 → 알림 수신
```

- 신청이 접수되면 상태가 **심사중**으로 변경됩니다.
- 관리자가 심사 후 **승인** 또는 **반려** 처리합니다.
- 결과는 알림(🔔)으로 안내됩니다.

### 4.3 주의사항

- 동일 Entity에서 같은 앱을 중복 신청할 수 없습니다.
- Entity Code는 영문, 숫자, 하이픈(`-`)만 사용할 수 있습니다.
- 이미 사용 중이거나 심사 중인 앱은 재신청 버튼이 표시되지 않습니다.

## 5. 내 구독 관리 (Managing My Subscriptions)

### 5.1 내 구독 페이지 접근

상단 헤더의 **[📋 내 구독 관리]** 메뉴를 클릭하면 `/my-subscriptions` 페이지로 이동합니다.

### 5.2 필터 탭

구독 목록 상단에서 상태별 필터 탭을 선택할 수 있습니다:

| 탭 | 포함 상태 | 설명 |
|----|-----------|------|
| **전체** | 모든 상태 | 전체 구독 목록 표시 |
| **심사중** | PENDING | 관리자 승인 대기 중 |
| **사용중** | ACTIVE | 현재 사용 중인 앱 |
| **만료** | EXPIRED | 구독 기간이 만료된 앱 |
| **거절·취소** | REJECTED, CANCELLED, SUSPENDED | 거절/취소/정지된 앱 |

### 5.3 구독 카드 정보

각 구독 카드에 표시되는 정보:

| 항목 | 설명 |
|------|------|
| **앱 아이콘 + 이름** | 구독 중인 앱 |
| **Entity 정보** | Entity Code · Entity Name |
| **상태 뱃지** | 현재 구독 상태 (색상 구분) |
| **신청일** | 구독을 신청한 날짜 |
| **만료일** | 구독 만료 예정일 (무기한인 경우 "무기한" 표시) |
| **거절 사유** | 반려된 경우 사유 표시 (빨간 박스) |

### 5.4 구독 상태별 색상

| 상태 | 뱃지 색상 | 설명 |
|------|-----------|------|
| **심사중** | 🟡 노랑 | 관리자 심사 대기 |
| **사용중** | 🟢 초록 | 정상 사용 중 |
| **정지됨** | 🟣 보라 | 관리자에 의해 일시 정지 |
| **거절됨** | 🔴 빨강 | 신청이 반려됨 |
| **취소됨** | ⚪ 회색 | 사용자/관리자에 의해 취소됨 |
| **만료됨** | 🟠 주황 | 구독 기간 만료 |

### 5.5 구독 카드 액션 버튼

| 상태 | 가능한 액션 | 설명 |
|------|-------------|------|
| **사용중** | `앱 바로가기` + `구독 취소` | 앱으로 이동 또는 구독 취소 |
| **심사중** | `구독 취소` | 심사 대기 중 취소 |
| **거절됨** | `재신청` | 앱 상세 페이지로 이동하여 재신청 |
| **취소됨** | `재신청` | 앱 상세 페이지로 이동하여 재신청 |
| **만료됨** | `재신청` | 앱 상세 페이지로 이동하여 재신청 |
| **정지됨** | - | 관리자에게 문의 필요 |

### 5.6 구독 취소하기

1. 내 구독 페이지에서 취소할 앱의 **[구독 취소]** 버튼을 클릭합니다.
2. "정말 이 앱 구독을 취소하시겠습니까?" 확인 메시지가 표시됩니다.
3. **[확인]** 을 클릭하면 구독이 즉시 취소됩니다.

> **주의**: 취소된 구독은 복원할 수 없습니다. 다시 사용하려면 재신청이 필요합니다.

### 5.7 빈 상태

신청한 앱이 없으면 안내 메시지와 **[앱 둘러보기]** 버튼이 표시됩니다.

## 6. 알림 (Notifications)

### 6.1 알림 벨

상단 헤더의 🔔 아이콘을 통해 알림을 확인할 수 있습니다.

- **빨간 숫자 뱃지**: 읽지 않은 알림 수 (최대 `99+`)
- **뱃지 없음**: 미읽음 알림 없음
- 미읽음 수는 **30초마다** 자동 갱신됩니다.

### 6.2 알림 드롭다운

🔔 아이콘을 클릭하면 최근 알림 5건이 드롭다운으로 표시됩니다.

| 표시 항목 | 설명 |
|-----------|------|
| **알림 아이콘** | 알림 유형에 따른 아이콘 (아래 표 참조) |
| **제목** | 알림 제목 (미읽음: 굵은 글씨) |
| **내용** | 알림 상세 내용 (1줄) |
| **시간** | 상대 시간 (방금 전, 5분 전, 3시간 전, 2일 전 등) |
| **읽음 표시** | 미읽음 알림에 파란 점 표시 |

### 6.3 알림 유형

| 유형 | 아이콘 | 색상 | 발생 시점 |
|------|--------|------|-----------|
| **구독 승인** | ✓✓ | 🟢 초록 | 관리자가 구독을 승인했을 때 |
| **구독 반려** | ✕ | 🔴 빨강 | 관리자가 구독을 반려했을 때 |
| **구독 정지** | ⚠ | 🟡 노랑 | 관리자가 구독을 정지했을 때 |
| **구독 만료** | 🕐 | ⚪ 회색 | 구독 기간이 만료되었을 때 |
| **만료 임박** | ⚠ | 🟠 주황 | 구독 만료 7일 전 |
| **시스템** | 🔔 | 🔵 파랑 | 시스템 공지사항 |

### 6.4 알림 관리

| 기능 | 설명 |
|------|------|
| **개별 읽음** | 알림 항목 클릭 시 읽음 처리 |
| **전체 읽음** | 드롭다운 상단 `전체 읽음` 클릭 시 모든 알림 읽음 처리 |

> **참고**: 알림 드롭다운 외부를 클릭하면 드롭다운이 자동으로 닫힙니다.

## 8. 관리자 콘솔 (Admin Console)

> 이 섹션은 **ADMIN 역할**이 부여된 사용자만 해당됩니다.

### 8.1 관리자 콘솔 접근

로그인한 사용자에게 ADMIN 역할이 있으면 상단 헤더에 **[⚙ Admin]** 링크가 표시됩니다.

- 클릭 시 `/admin/subscriptions` (구독 관리) 페이지로 이동합니다.
- 별도로 `/admin/login` 페이지에서 JWT 토큰으로 로그인할 수도 있습니다.

### 8.2 관리자 사이드바 메뉴

| 메뉴 | 경로 | 설명 |
|------|------|------|
| **구독 관리** | `/admin/subscriptions` | 전체 구독 신청 관리 |
| **앱 관리** | `/admin/apps` | 앱 등록/수정/삭제 |
| **통계** | `/admin/stats` | 구독 통계 대시보드 |
| **스토어로 돌아가기** | `/` | 일반 사용자 화면으로 복귀 |

### 8.3 구독 관리 (Subscription Management)

구독 관리 페이지에서는 전체 앱에 대한 모든 구독 신청을 관리합니다.

#### 검색 및 필터

| 필터 | 설명 |
|------|------|
| **검색** | Entity Code, 이름, 이메일로 검색 |
| **상태 필터** | 대기 / 활성 / 정지 / 반려 / 해제 / 만료 |
| **앱 필터** | 특정 앱의 구독만 조회 |

#### 구독 목록 테이블

| 컬럼 | 설명 |
|------|------|
| **Entity** | Entity Code + Entity Name |
| **앱** | 구독 대상 앱 이름 |
| **신청자** | 신청자 이름 + 이메일 |
| **상태** | 상태 뱃지 (색상 구분) |
| **만료일** | 만료 예정일 (미설정 시 `-`) |
| **신청일** | 신청 접수 날짜 |
| **관리** | 상태에 따른 액션 버튼 |

#### 상태별 관리 액션

| 현재 상태 | 가능한 액션 | 설명 |
|-----------|-------------|------|
| **대기** (PENDING) | `승인` `반려` | 신규 신청 심사 |
| **활성** (ACTIVE) | `정지` `해제` | 사용 중인 구독 관리 |
| **정지** (SUSPENDED) | `재활성화` `해제` | 정지된 구독 복원 또는 완전 해제 |
| **만료** (EXPIRED) | `재활성화` | 만료된 구독 재활성화 |
| **반려** (REJECTED) | - | 추가 액션 없음 |
| **해제** (CANCELLED) | - | 추가 액션 없음 |

#### 승인 처리

1. 대기 상태의 구독에서 **[승인]** 버튼을 클릭합니다.
2. 승인 모달이 열립니다.
3. **만료일 설정** (선택사항):
   - 비워두면 **무기한** 구독이 됩니다.
   - 날짜를 선택하면 해당 날짜에 구독이 자동 만료됩니다.
4. **[승인]** 버튼을 클릭합니다.

#### 반려 처리

1. 대기 상태의 구독에서 **[반려]** 버튼을 클릭합니다.
2. 반려 모달이 열립니다.
3. **반려 사유**를 입력합니다 (필수).
4. **[확인]** 버튼을 클릭합니다.

> **참고**: 반려 사유는 사용자의 "내 구독 관리" 페이지에서 확인할 수 있습니다.

## 9. 자주 묻는 질문 (FAQ)

### Q1. 구독 신청 후 얼마나 걸리나요?

관리자가 신청 내용을 확인한 후 승인/반려 처리합니다. 일반적으로 1~2 영업일 이내 처리되며, 결과는 알림으로 안내됩니다.

### Q2. 구독이 만료되면 어떻게 되나요?

구독 만료 7일 전에 알림이 발송됩니다. 만료일이 지나면 자동으로 **만료됨** 상태로 전환되며, 앱 접근이 제한됩니다. "내 구독 관리" 페이지에서 **[재신청]** 버튼을 통해 다시 신청할 수 있습니다.

### Q3. 구독을 취소했는데 다시 사용하고 싶어요.

취소 후에는 "내 구독 관리" 페이지에서 **[재신청]** 버튼을 클릭하여 앱 상세 페이지에서 다시 신청하면 됩니다.

### Q4. "이미 신청 중이거나 사용 중인 앱입니다" 메시지가 나옵니다.

동일 Entity에서 같은 앱에 대해 이미 활성 구독이 있거나 심사 중인 신청이 있는 경우 발생합니다. "내 구독 관리" 페이지에서 기존 구독 상태를 확인해 주세요.

### Q5. 구독이 정지되었습니다.

관리자에 의해 일시적으로 정지된 상태입니다. 관리자에게 문의하여 사유를 확인하세요.

### Q6. 반려 사유는 어디서 확인하나요?

"내 구독 관리" 페이지에서 거절된 구독 카드에 빨간색 박스로 **거절 사유**가 표시됩니다.

### Q7. 다국어 지원은 어떻게 되나요?

상단 헤더의 🌐 언어 버튼을 통해 한국어, English, Tiếng Việt으로 전환할 수 있습니다. 선택한 언어는 브라우저에 자동 저장됩니다.

### Q8. 앱 사용 중 "접근 권한 없음" 오류가 발생합니다.

- 구독이 만료되었거나 정지/취소되었을 수 있습니다.
- "내 구독 관리" 페이지에서 구독 상태를 확인하세요.
- 구독이 활성 상태인데도 오류가 발생하면 로그아웃 후 재로그인을 시도하세요.
