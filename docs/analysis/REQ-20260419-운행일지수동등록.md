## 2. AS-IS 현황 분석

### 2.1 Backend

| 항목 | 현황 |
|------|------|
| Controller | `trip-log.controller.ts` — `GET /`, `GET /:id`, `PATCH /:id`, `PATCH /:id/submit`, `POST /import` |
| Service | `createFromDispatch()` — 배차 생성 시 자동 호출, `POST` 엔드포인트 없음 |
| Entity | `TripLogEntity` — `cdr_id` (배차요청 FK) **NOT NULL** → 수동 생성 시 proxy dispatch 필요 |
| Import 패턴 | `import-orchestrator.service.ts`에서 `cdrIsProxy=true` proxy dispatch 생성 후 trip log 연결 |

**문제점**: 수동 운행일지 등록 API가 없음. 현재 운행일지는 배차(dispatch) 연동 또는 Excel 업로드로만 생성됨.

### 2.2 Frontend

| 항목 | 현황 | 파일 경로 |
|------|------|-----------|
| 페이지 | `TripLogFormPage.tsx` — stub (placeholder만 표시) | `pages/TripLogFormPage.tsx` |
| 라우터 | `/trip-logs/new` 라우트 등록 완료 | `App.tsx` |
| API | `tripLogApi` — `create` 메서드 없음 | `services/api.ts` |
| Hook | `useTripLogs.ts` — `useCreateTripLog` 없음 | `hooks/useTripLogs.ts` |
| i18n | `tripLog` 섹션 — 폼 관련 키 부족 (`selectVehicle`, `selectDriver`, `customerName` 등 없음) | `i18n/locales/*/car.json` |

### 2.3 DB

| 테이블 | 관련 컬럼 | 비고 |
|--------|-----------|------|
| `car_trip_logs` | `cdr_id` CHAR(36) NOT NULL | proxy dispatch 생성 필수 |
| `car_dispatch_requests` | `cdr_is_proxy` BOOLEAN | Excel import에서 이미 사용 중 |

## 5. 사용자 플로우

```
1. 사용자 → /trip-logs/new 접속
2. 차량 드롭다운에서 차량 선택
3. 운전자 드롭다운에서 운전자 선택
4. 출발지/목적지 입력 (필수)
5. 운행 정보 입력 (출발일시, 도착일시, 주행거리 등)
6. 비용 정보 입력 (주유/통행료 등)
7. [저장] 클릭
8. Backend: proxy dispatch 생성 → trip log 생성
9. 성공 시 → /trip-logs 목록으로 이동
```

---

## 6. 기술 제약사항

- `cdr_id` NOT NULL → proxy DispatchRequest 자동 생성 필수 (import-orchestrator 패턴 참고)
- 차량/운전자 목록은 기존 API (`GET /v1/vehicles`, `GET /v1/drivers`) 활용
- Form validation: React Hook Form + Zod (프로젝트 표준)
