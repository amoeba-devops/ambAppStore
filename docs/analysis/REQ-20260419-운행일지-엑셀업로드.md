## 2. 스키마 확장 필요 항목

### 2.1 car_trip_logs 컬럼 추가

```sql
ALTER TABLE car_trip_logs
  ADD COLUMN ctl_customer_name VARCHAR(100) NULL AFTER ctl_destination,
  ADD COLUMN ctl_bill_no       VARCHAR(100) NULL AFTER ctl_customer_name,
  ADD COLUMN ctl_cdf_no        VARCHAR(200) NULL AFTER ctl_bill_no;
```

### 2.2 car_trip_log_fees 신규 테이블

```sql
CREATE TABLE car_trip_log_fees (
  ctlf_id       CHAR(36)    NOT NULL PRIMARY KEY,
  ctl_id        CHAR(36)    NOT NULL,
  ent_id        CHAR(36)    NOT NULL,
  ctlf_type     ENUM('FUEL','TOLL','PARKING','WASH','TIRE','MAINTENANCE','OTHER') NOT NULL,
  ctlf_amount   INT         NOT NULL,
  ctlf_currency CHAR(3)     NOT NULL DEFAULT 'VND',
  ctlf_note     VARCHAR(200) NULL,
  ctlf_created_at DATETIME  NOT NULL DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_ctlf_ctl (ctl_id),
  FOREIGN KEY (ctl_id) REFERENCES car_trip_logs(ctl_id)
);
```

### 2.3 car_import_logs 신규 테이블

```sql
CREATE TABLE car_import_logs (
  cil_id          CHAR(36)    NOT NULL PRIMARY KEY,
  ent_id          CHAR(36)    NOT NULL,
  cil_filename    VARCHAR(255) NOT NULL,
  cil_profile     VARCHAR(50)  NOT NULL,
  cil_total_rows  INT          NOT NULL DEFAULT 0,
  cil_success_cnt INT          NOT NULL DEFAULT 0,
  cil_fail_cnt    INT          NOT NULL DEFAULT 0,
  cil_status      ENUM('PROCESSING','DONE','FAILED') NOT NULL DEFAULT 'PROCESSING',
  cil_uploaded_by CHAR(36)    NOT NULL,
  cil_created_at  DATETIME    NOT NULL DEFAULT CURRENT_TIMESTAMP,
  cil_updated_at  DATETIME    NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_cil_ent (ent_id)
);
```

### 2.4 cdr_id NULL 허용 변경

`car_trip_logs.cdr_id` → Synthetic Dispatch 방식 채택 (NOT NULL 유지).
Import 시 배차 레코드를 `status=COMPLETED, cdr_note='IMPORTED_FROM_EXCEL'`로 자동 생성.

## 3. 필드 매핑 매트릭스 (CR-Vietnam-Truck-v1)

| # | 엑셀 원본 | 변환 규칙 | 매핑 대상 |
|---|----------|----------|----------|
| 1 | A(NO) | 미사용 | — |
| 2 | B(DATE)+C(START) | YYYY-MM-DD + HhMM → DATETIME | `ctl_depart_actual` |
| 3 | B(DATE)+D(FINISH) | 동일 / 자정 넘김 +1일 보정 | `ctl_arrive_actual` |
| 4 | E(FROM) | trim·lower | `ctl_origin` |
| 5 | F(TO) | trim·lower | `ctl_destination` |
| 6 | G(START KM) | 그대로 | `ctl_odo_start` |
| 7 | H(FINISH KM) | 그대로 | `ctl_odo_end` |
| 8 | I(TOTAL KM) | 없으면 H-G 계산 | `ctl_distance_km` |
| 9 | J(CUSTOMER) | UPPER·TRIM | `ctl_customer_name` (신규) |
| 10 | K(BILL/BOOK) | trim | `ctl_bill_no` (신규) |
| 11 | L(CDF) | trim | `ctl_cdf_no` (신규) |
| 12 | M(TOLL) | 그대로 | `ctl_toll_cost` + fee 레코드 |
| 13 | N+O(XANG) | 주유비 + refueled=TRUE | `ctl_fuel_cost`, `ctl_refueled` |
| 14 | N+O(BAI/RUA XE 등) | 분류별 fee 레코드 | `car_trip_log_fees` |
| 15 | P(단가) | 참조용 저장 | `ctl_note` 또는 별도 |
| 16 | Q(소비량) | 추정치 보관 | `ctl_fuel_amount` |

## 5. 기술 제약사항

| 항목 | 설명 |
|------|------|
| 파일 형식 | `.xls` (BIFF8) + `.xlsx` (OOXML) — exceljs 라이브러리 |
| 파일 크기 | 최대 10MB |
| 처리 시간 | 1,000행 기준 60초 이내 |
| 트랜잭션 | 행 단위 (부분 실패 허용 + 이력표) |
| 문자셋 | UTF-8 / 베트남어 성조 보존 |
| 중복 방지 | 멱등키: (cvh_id + ctl_depart_actual + odo_start) |
