## 5. 사용자 플로우 / User Flow

### 5.1 AMA → AppStore iframe 접속 플로우

```
AMA Entity Settings 페이지
  └─ "앱스토어" 메뉴 클릭
     └─ iframe src="https://stg-apps.amoeba.site/" + JWT token (URL param or postMessage)
        └─ AppStore 로드
           ├─ isEmbedded() = true 감지
           ├─ JWT 토큰 → auth.store 저장 → user 정보 복원
           └─ Header 렌더링
              └─ [VN01 · Company Name] [🌐 한국어]
```

### 5.2 구독 신청 플로우 (모달)

```
앱 상세 페이지 (/apps/app-car-manager)
  └─ "앱 사용 신청" 버튼 클릭
     └─ SubscriptionRequestModal 열림
        ├─ 화면 내 스크롤 (max-h-[90vh])
        ├─ 자동 입력: Entity ID, Code (토큰 기반)
        ├─ 수동 입력: Entity Name
        ├─ 선택 입력: 신청 사유
        └─ "신청하기" 버튼 → API 호출
           ├─ 성공 → 성공 메시지
           └─ 실패 → 에러 표시
```

## 6. 기술 제약사항 / Technical Constraints

| # | 항목 | 설명 |
|---|------|------|
| C-01 | Cross-origin iframe | AMA(ama.amoeba.site) → AppStore(stg-apps.amoeba.site) cross-origin → `document.referrer`는 접근 가능 |
| C-02 | X-Frame-Options | Nginx `SAMEORIGIN` 설정 → AMA iframe 허용을 위해 `ALLOW-FROM` 또는 CSP `frame-ancestors` 필요 |
| C-03 | JWT 전달 | iframe 내 JWT 전달: URL param `?token=` 또는 `postMessage` → 현재 `localStorage('ama_token')` 방식 유지 |
| C-04 | 하위 호환 | 기존 public 구독 API에서 `requester_name`, `requester_email`이 optional이므로 백엔드 변경 불필요 |
