## 5. 사용자 플로우

```
[차량 목록] 
    ↓ 차량 row 클릭
[차량 상세 페이지 /vehicles/:id]
    ↓ "수정" 버튼 클릭
[차량 수정 페이지 /vehicles/:id/edit]
    ↓ (데이터 로딩: useVehicle)
    │
    ├─ 로딩 중 → 로딩 인디케이터 표시
    │
    └─ 로딩 완료 → 폼 렌더링 (기존 값 prefill)
            ↓ 사용자 편집
            ↓ "저장" 버튼 클릭
            ↓ PATCH /v1/vehicles/:id
            │
            ├─ 성공 → toast "저장됨" → navigate(/vehicles/:id)
            │         (캐시 자동 invalidate → 상세 페이지 새 값 반영)
            │
            └─ 실패 → toast 에러 메시지 → 페이지 유지
    
    취소 버튼 클릭 시 → navigate(/vehicles/:id) (변경 폐기)
```

### 5.1 조건별 분기

| 조건 | 동작 |
|------|------|
| 잘못된 ID (404) | `useVehicle` 훅이 error → "차량을 찾을 수 없습니다" 표시 |
| 다른 법인 차량 접근 | 백엔드 `ent_id` 검증으로 404 반환 → 동일 처리 |
| 수정 권한 없음 | (현 단계에선 모든 구독 사용자 허용, 추후 권한 분리 시 확장) |
| validation 실패 | 필드별 에러 메시지 표시, 저장 버튼 disabled |
