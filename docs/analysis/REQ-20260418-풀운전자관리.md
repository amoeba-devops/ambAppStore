## 2. AS-IS 현황 분석

### 2.1 풀 운전자 개념 (reference/req/AMA-VEH-REQ-1.1.0.md)

```
POOL_DRIVER — 법인 내 모든 차량 운행 가능, 차량 미고정
             여러 차량에 공유 배정되는 운전자
             차량별 배정/해제 가능
```

### 2.2 Backend — 완전 구현됨

| API | 용도 | 상태 |
|-----|------|------|
| `GET /drivers?status=ACTIVE` | 전체 운전자 조회 | ✅ |
| `GET /drivers/available` | 가용 운전자 목록 | ✅ |
| `PATCH /drivers/:id/assign` | 차량 배정 (`{ vehicle_id }`) | ✅ |
| `PATCH /drivers/:id/unassign` | 차량 해제 (cvhId → null) | ✅ |

### 2.3 Frontend — 미구현

| 항목 | 상태 |
|------|------|
| "풀 운전자 관리" 버튼 (VehicleDetailPage:127-129) | ⚠️ 껍데기 — onClick 없음 |
| 풀 운전자 관리 모달 | ❌ 미구현 |
| `driverApi.assign()` / `driverApi.unassign()` | ✅ 정의됨 (미사용) |

### 2.4 현재 운전자 탭 표시 구조

```
차량 상세 → 운전자 탭
├── ★ 주운전자 (PRIMARY_DRIVER) — 이 차량에 배정된 주운전자
├── 부운전자 (SUB_DRIVER) — 이 차량에 배정된 부운전자
└── 풀 운전자 (POOL_DRIVER) — 이 차량에 배정된 풀 운전자
    ※ 현재는 이 차량에 직접 배정된 풀 운전자만 표시
    ※ 법인 내 다른 풀 운전자를 배정/해제하는 UI 없음
```

## 3. TO-BE 요구사항

### 3.1 R-001: 풀 운전자 관리 모달 (`PoolDriverModal`)

**트리거**: "풀 운전자 관리" 버튼 클릭

**모달 구성:**
```
┌────────────────────────────────────────┐
│  🌐 풀 운전자 관리                      │
│                                        │
│  ── 이 차량에 배정된 풀 운전자 ──       │
│  ┌──────────────────────────────┐      │
│  │ 김철수 (kim@co.kr)  [해제]   │      │
│  │ 이영희 (lee@co.kr)  [해제]   │      │
│  └──────────────────────────────┘      │
│                                        │
│  ── 배정 가능한 풀 운전자 ──            │
│  ┌──────────────────────────────┐      │
│  │ 박민수 (park@co.kr) [배정]   │      │
│  │ 최수진 (choi@co.kr) [배정]   │      │
│  └──────────────────────────────┘      │
│                                        │
│  [닫기]                                │
└────────────────────────────────────────┘
```

- **상단**: 현재 차량에 배정된 `POOL_DRIVER` 목록 + "해제" 버튼
- **하단**: 차량 미배정(`cvhId = null`) 또는 다른 차량에 배정된 `POOL_DRIVER` 목록 + "배정" 버튼
- 배정/해제 즉시 반영 (목록 갱신)

### 3.2 R-002: 배정 (assign)

- "배정" 클릭 → `PATCH /drivers/:id/assign { vehicle_id: 현재차량ID }`
- 성공 → 해당 운전자가 "배정된" 목록으로 이동
- 차량 상세의 운전자 목록도 갱신

### 3.3 R-003: 해제 (unassign)

- "해제" 클릭 → `PATCH /drivers/:id/unassign`
- 성공 → 해당 운전자가 "가용" 목록으로 이동
