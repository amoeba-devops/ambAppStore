## 3. TO-BE 요구사항

### 3.1 R-001: 운전자 추가 모달 (`DriverFormModal`)

| 항목 | 설명 |
|------|------|
| 위치 | 차량 상세 페이지 → 운전자 탭 → "운전자 추가" 버튼 클릭 |
| 컴포넌트 | `DriverFormModal.tsx` |
| 트리거 | VehicleDetailPage "운전자 추가" 버튼 onClick |

**폼 필드:**

| 필드 | 유형 | 필수 | 설명 |
|------|------|------|------|
| AMA 사용자 ID | text input (UUID) | ✅ | `ama_user_id` — AMA 사용자 식별자 |
| 역할 | select | ✅ | PRIMARY_DRIVER / SUB_DRIVER / POOL_DRIVER |
| 차량 배정 | 자동 | - | 현재 차량 ID (`vehicle_id`) 자동 전달 |
| 메모 | textarea | - | 선택 입력 |

**동작 흐름:**
1. "운전자 추가" 버튼 클릭 → 모달 열림
2. AMA 사용자 ID 입력 + 역할 선택
3. "등록" 클릭 → `POST /api/v1/drivers` 호출
   - body: `{ ama_user_id, role, vehicle_id, note? }`
4. 성공 → 모달 닫기 + 운전자 목록 갱신 (queryClient.invalidateQueries)
5. 실패 → 에러 메시지 표시
   - `CAR-E4002`: 이미 등록된 운전자

### 3.2 R-002: 비즈니스 로직 (이미 백엔드 구현됨)

- 동일 법인 내 동일 AMA 사용자 중복 등록 방지
- 운전자 생성 시 차량 즉시 배정 가능 (vehicle_id 전달)
- POOL_DRIVER는 여러 차량에 공유 → vehicle_id 없이 등록 가능

### 3.3 R-003: DriverListPage 라우트 등록

- `App.tsx`에 `/drivers` 라우트 추가
- DriverListPage "운전자 등록" 버튼에 onClick → DriverFormModal 열기 (vehicle_id 없이)

### 3.4 R-004: 사이드바 네비게이션

- `AppLayout.tsx` nav sections에 "운전자 관리" 메뉴 추가
- 아이콘: `UserCheck` (lucide-react)
- 경로: `/drivers`

## 5. 사용자 플로우

### 5.1 차량 상세 → 운전자 추가

```
[차량 상세 페이지]
    │
    ├─ [운전자 탭] 클릭
    │
    ├─ [운전자 추가] 버튼 클릭
    │     │
    │     ▼
    │  ┌─────────────────────────┐
    │  │   DriverFormModal       │
    │  │                         │
    │  │  AMA 사용자 ID: [____]  │
    │  │  역할: [PRIMARY ▼]      │
    │  │  메모: [____]           │
    │  │                         │
    │  │  [취소]  [등록]         │
    │  └─────────────────────────┘
    │         │
    │    POST /api/v1/drivers
    │    { ama_user_id, role, vehicle_id, note? }
    │         │
    │    ┌────┴────┐
    │    │ 성공    │ 실패 (CAR-E4002: 중복)
    │    │         │
    │    ▼         ▼
    │  모달 닫기   에러 메시지 표시
    │  목록 갱신
    │
    ▼
[운전자 목록 갱신됨 — 역할별 그룹 표시]
```

### 5.2 운전자 목록 페이지 → 운전자 등록

```
[사이드바] → [운전자 관리] 클릭 → /drivers
    │
    ├─ [운전자 등록] 버튼 클릭
    │     │
    │     ▼
    │  DriverFormModal (vehicle_id 없이)
    │  → 역할 선택 + AMA 사용자 ID 입력
    │  → 등록 → 목록 갱신
```
