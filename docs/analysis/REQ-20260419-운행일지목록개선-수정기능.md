### 2.2 Backend — TripLog API

| 항목 | 현황 | 파일 경로 |
|------|------|-----------|
| findAll | `vehicle`, `driver` relation JOIN하나 매퍼에서 driver 정보 미매핑 | `service/trip-log.service.ts` |
| findAll | `dispatchRequest` relation 미JOIN → 탑승인원 조회 불가 | 위 동일 |
| Mapper | `vehiclePlateNumber`, `vehicleModel`만 매핑 → `driverName`, `passengerCount` 없음 | `mapper/trip-log.mapper.ts` |
| Response DTO | `driverName`, `passengerCount` 필드 없음 | `dto/response/trip-log.response.ts` |
| Update DTO | `driver_id` 필드 없음 → 배정운전자 변경 불가 | `dto/request/trip-log.request.ts` |
| Service.update | `driver_id` 매핑 없음 | `service/trip-log.service.ts` |

### 2.3 데이터 접근 경로

| 데이터 | 소스 | 현재 상태 |

## 3. TO-BE 요구사항

### 3.1 목록 컬럼 변경

**AS-IS**: 차량번호 | 날짜 | 노선 | 주행거리 | 주유비 | 상태
**TO-BE**: 날짜 | 차량번호 | 노선 | 주행거리 | 상태

- `주유비` 컬럼 제거, `날짜`를 첫 번째로 이동

### 3.2 정렬/필터

| 필터 | 방식 |
|------|------|
| 날짜 | 기존 월 선택기 유지 (client-side) |
| 차량번호 | 드롭다운 필터 (전체 + 등록된 차량 목록) |
| 노선 | 텍스트 검색 (출발지/목적지 포함 검색, client-side) |

### 3.3 인라인 수정 기능

운행일지 행 확장 시 **수정 버튼** 표시 → 클릭 시 인라인 편집 모드 전환:

| 수정 필드 | 타입 | 비고 |
|-----------|------|------|
| 배정운전자 | 드롭다운 (운전자 목록) | driver_id PATCH |
| 탑승인원 | 숫자 입력 | dispatchRequest.cdrPassengerCount PATCH |
| 출발시 거리계 | 숫자 입력 | odo_start |
| 도착시 거리계 | 숫자 입력 | odo_end |
| 주행거리 | 자동 계산 (도착-출발) | distance_km 자동 |
| 주유량 | 숫자 입력 | fuel_amount |
| 통행료 | 숫자 입력 | toll_cost |

- VERIFIED 상태인 운행일지는 수정 불가 (기존 로직 유지)
- 저장 시 `PATCH /api/v1/trip-logs/:id` 호출

### 3.4 Backend 변경

| 변경 | 내용 |
|------|------|
| TripLogResponse | `driverName`, `passengerCount` 필드 추가 |
| TripLogMapper | driver.cvdDriverName, dispatchRequest.cdrPassengerCount 매핑 |
| findAll | dispatchRequest relation 추가 JOIN |
| UpdateTripLogRequest | `driver_id` (UUID, optional) 필드 추가 |
| Service.update | `driver_id` → `cvdId` 매핑 추가 |

## 5. 사용자 플로우

### 5.1 목록 조회

```
1. /trip-logs 접속
2. 목록: 날짜 | 차량번호 | 노선 | 주행거리 | 상태 표시
3. 월 선택기로 기간 필터
4. 차량번호 드롭다운으로 차량 필터
5. 노선 검색으로 출발지/목적지 검색
```

### 5.2 운행일지 수정

```
1. 운행일지 행 클릭 → 상세 펼침
2. [수정] 버튼 클릭 → 인라인 편집 모드
3. 필드 수정 (배정운전자, 탑승인원, 거리계, 주유량, 통행료)
4. [저장] 클릭 → PATCH API 호출
5. 성공 시 목록 갱신, 편집 모드 종료
6. [취소] 시 원래 값 복원
```

## 6. 기술 제약사항

- VERIFIED 상태 운행일지는 수정 불가 (기존 비즈니스 규칙)
- 탑승인원 수정은 dispatchRequest 테이블 업데이트 필요 — trip log update API에서 처리 또는 별도 dispatch PATCH 사용
- 주행거리 = 도착거리계 - 출발거리계 자동 계산 (기존 로직 유지)
- dispatchRequest JOIN 추가 시 N+1 쿼리 주의 → QueryBuilder leftJoinAndSelect 사용
